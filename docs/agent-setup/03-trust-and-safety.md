# Agent Setup — Trust & Safety Agent

> Facilitator reference: how the **Trust & Safety Agent** at [aka.ms/a2a2i](https://aka.ms/a2a2i) was built. Use this if you want to recreate it in your own tenant, adapt it for a different audience, or understand why it behaves the way it does in the demo.

---

## Purpose

Take a **summary of a built agent** (generated by the agent itself in Copilot Agent Builder) and produce a lightweight, structured **Agent Charter** covering scope, guardrails, ownership, and success signals.

This is the *third and final* step in the A2A2I flow. It exists so participants leave the workshop with not just an agent, but a clear, responsible record of *what they built and why*.

### Starter prompts

1. *Here's a summary of an agent I just built — generate its Agent Charter.*
2. *Help me document my agent for responsible rollout.*
3. *Write the scope and guardrails for this agent.*
4. *Turn this agent description into a one-page charter.*

### Knowledge sources

- **None.** The user pastes the self-summary from their built agent.

Key behaviours the prompt enforces:

- Expects an agent **self-summary** as input — prompts the user if it's missing
- Outputs a **structured Agent Charter** with consistent sections: purpose, intended users, in-scope, out-of-scope, guardrails / refusals, data handling, owner, success signals, review cadence
- Uses **plain, non-jargon language** suitable for nonprofit / public-sector audiences
- Flags obvious risks (sensitive data, decisions that need human review, scope creep) but does **not** block — this is a coaching tool, not a gatekeeper
- Stays grounded in what the user pasted — does not invent capabilities the agent doesn't have

## How to recreate it

1. In [Microsoft 365 Copilot](https://m365.cloud.microsoft/chat), click **New agent** → **Skip to configure**
2. Set the identity fields above
3. Paste the system prompt into **Instructions**
4. Add the four starter prompts
5. Leave **Knowledge** empty
6. **Save** and **Publish** (or share via link)
7. Test using the [sample transcript](../sample-transcripts/trust-and-safety.md) as a smoke test
